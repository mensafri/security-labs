Modul Pembelajaran Lengkap: Kriptografi & Keamanan Aplikasi

MODUL 1: Kriptografi Dasar

1. Pengantar

Kriptografi dibutuhkan untuk melindungi data sensitif (password, token, data pribadi) dari ancaman seperti penyadapan (eavesdropping), modifikasi (tampering), dan pemalsuan.

Tujuan Kriptografi (CIA Triad):
  • Confidentiality (Kerahasiaan)
  • Integrity (Keutuhan)
  • Authentication (Otentikasi)
  • Non-repudiation (Nirpenyangkalan)

2. Enkripsi Simetris

Menggunakan satu kunci yang sama untuk enkripsi dan dekripsi. Contoh: AES (Advanced Encryption Standard).

Kelebihan: Cepat. Kekurangan: Distribusi kunci sulit.

3. Enkripsi Asimetris

Menggunakan dua kunci: Public Key (untuk enkripsi) dan Private Key (untuk dekripsi).

Contoh: RSA, ECC. Digunakan untuk pertukaran kunci aman.

4. Hubungan Hibrida

Kombinasi keduanya: Enkripsi Asimetris untuk Handshake (pertukaran kunci awal), lalu Enkripsi Simetris untuk transfer data selanjutnya (agar cepat).

MODUL 2: Hash & TLS

1. Hash Function

Fungsi satu arah yang mengubah input menjadi string panjang tetap (hash value). Tidak bisa didekripsi.

Karakteristik: Deterministik, Avalanche Effect, Collision Resistance.

2. Password Security

Simpan password dalam bentuk Hash, jangan plaintext! Gunakan Salt (data acak tambahan) untuk mencegah serangan Rainbow Table.

Algoritma Aman: bcrypt, Argon2, SHA-256 (dengan salt).

Algoritma Usang: MD5, SHA-1.

3. Digital Signature

Menjamin autentikasi dan integritas. Menggunakan Hash pesan yang dienkripsi dengan Private Key pengirim.

4. TLS / HTTPS

Protokol keamanan web. Menggunakan sertifikat digital (CA) untuk verifikasi identitas server.

Ancaman tanpa TLS:

Man-in-the-Middle (MITM) attack, di mana hacker di tengah jalur komunikasi bisa membaca data kita.
